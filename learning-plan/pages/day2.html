Q: 为什么使用全局单例而非依赖注入？

                    
A: Claude Code 选择全局单例模式主要基于以下考虑：

                    
    
                        
  • CLI 场景特殊性：CLI 是单进程、单会话的，不存在 Web 服务的并发请求隔离需求
    • 
                        
  • 启动性能：全局状态通过 memoize() 缓存，避免重复初始化
    • 
                        
  • 跨模块共享：会话 ID、工作目录、配置等需要在任意位置访问
    • 
                        
  • 状态持久化：使用 bootstrap/state.ts 集中管理，便于调试和序列化
    • 
                    

                    
代码中明确注释 "DO NOT ADD MORE STATE HERE - BE JUDICIOUS WITH GLOBAL STATE"，强调对全局状态的审慎态度。

                

                

                    
Q: 初始化过程中有哪些性能瓶颈？

                    
A: 根据源码分析，主要瓶颈包括：

                    
    
                        
  • 配置验证：enableConfigs() 需要读取并解析多个配置文件
    • 
                        
  • OAuth 令牌获取：macOS Keychain 读取是同步阻塞操作（~65ms）
    • 
                        
  • MTLS 配置：TLS 证书加载必须在第一次 TLS 握手前完成
    • 
                        
  • OpenTelemetry 初始化：延迟加载以避免 ~400KB 的模块开销
    • 
                    

                    
优化策略：使用 startKeychainPrefetch() 并行预取、applySafeConfigEnvironmentVariables() 分阶段加载配置。
