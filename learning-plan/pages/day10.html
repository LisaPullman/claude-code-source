Q: 子代理与主代理如何通信？

                    
A: AgentTool 的通信架构：

                    
    
                        
  • 上下文传递：主代理通过 createSubagentContext() 创建子代理上下文
    • 
                        
  • 消息流：子代理的输出通过 yield* 流式返回主代理
    • 
                        
  • 状态隔离：每个子代理有独立的 abortController 和 readFileState
    • 
                        
  • 进度报告：AgentToolProgress 类型定义进度更新格式
    • 
                        
  • 颜色标识：agentColorManager.ts 为不同代理分配显示颜色
    • 
                    

                    
内存管理：agentMemory.ts 和 agentMemorySnapshot.ts 管理代理间状态持久化。

                

                

                    
Q: 技能系统的扩展性如何？

                    
A: 技能系统支持多层扩展：

                    
    
                        
  • 文件系统技能：.claude/skills/*.md 目录中的 Markdown 文件
    • 
                        
  • MCP 技能：通过 MCP 服务器动态提供技能
    • 
                        
  • 内置技能：built-in/ 目录中的预定义技能
    • 
                        
  • 热重载：loadAgentsDir.ts 监听文件变化自动刷新
    • 
                        
  • 技能发现：dynamicSkillDirTriggers 按需加载技能目录
    • 
                    

                    
技能本质是预定义的提示词模板，通过 SkillTool.ts 注入到对话上下文。
