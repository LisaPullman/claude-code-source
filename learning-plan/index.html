📋 学习建议

学习方法

  • • 先整体后局部：先理解架构，再深入具体实现
  • • 边读边画图：流程图、时序图、架构图能加深理解
  • • 关注设计决策：不仅是"怎么做"，更要理解"为什么这么做"
  • • 实际运行项目：观察日志输出，验证理解
  • • 做笔记：每天记录关键发现和疑问

推荐工具

  • • VS Code 或类似 IDE 进行代码导航
  • • Draw.io 或 Excalidraw 绘制图表
  • • rg 或 IDE 搜索功能追踪函数调用链
  • • git log 查看重要文件的提交历史
