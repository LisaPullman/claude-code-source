# Claude Code 源代码 30 天学习计划

> 本项目是 Anthropic 开发的 agentic 编程工具 Claude Code（代号 Tengu），运行在终端中，通过自然语言命令帮助开发者理解代码库、执行常规任务、解释复杂代码以及处理 git 工作流。

---

## 学习前置准备

### 环境要求
- Node.js >= 18.0.0
- 熟悉 TypeScript / React
- 了解终端 UI 框架 Ink（React for CLI）

### 核心依赖库（先了解）
| 库 | 用途 |
|---|---|
| Commander.js | CLI 参数解析 |
| Ink + React | 终端 UI 渲染 |
| @anthropic-ai/sdk | Claude API 客户端 |
| @modelcontextprotocol/sdk | MCP 协议实现 |
| Zod v4 | 数据校验 |
| OpenTelemetry | 遥测与追踪 |
| GrowthBook | 特性开关管理 |

## 第一阶段：项目概览与架构理解（第 1-5 天）

### 第 1 天：项目结构与入口分析

**目标**：理解项目整体结构和启动流程

**阅读文件**：
1. `package.json` - 了解依赖、入口、引擎要求
2. `README.md` - 项目概述和使用说明
3. `src/entrypoints/cli.tsx` - **主要引导入口**，快速路径分发器

**学习要点**：
- `cli.tsx` 如何使用 `feature()` 进行编译时特性开关
- 快速路径模式（fast-path pattern）：`--version` 零导入优化
- 动态导入 `await import()` 的懒加载策略
- 启动性能分析 `profileCheckpoint()` 的使用

**思考题**：
- 为什么入口文件要使用快速路径模式？
- 特性开关如何影响打包体积？

### 学习建议

1. **先整体后局部**：先理解架构，再深入具体实现
2. **边读边画图**：流程图、时序图、架构图能加深理解
3. **关注设计决策**：不仅是"怎么做"，更要理解"为什么这么做"
4. **实际运行项目**：观察日志输出，验证理解
5. **做笔记**：每天记录关键发现和疑问

### 推荐学习工具

- 使用 VS Code 或类似 IDE 进行代码导航
- 使用 Draw.io 或 Excalidraw 绘制图表
- 使用 `rg` 或 IDE 搜索功能追踪函数调用链
- 使用 `git log` 查看重要文件的提交历史
